## 1. 프로젝트 한 줄 소개

Las Vegas 외식 시장에서 기존 상권 분석이 포착하지 못했던 "프랜차이즈 경쟁 압력"을 소비자 행동 데이터로 정량화하는 지수(FPI)를 직접 설계하고, 이를 기반으로 독립 브랜드의 생존·도태 패턴을 규명하여 입지 전략과 운영 전략을 데이터로 도출했습니다.

"프랜차이즈 옆 가게가 힘들다"는 직관을 통계로 입증하는 데 그치지 않고, 고압력 환경에서도 살아남는 브랜드가 소비자로부터 어떤 언어적 반응을 이끌어내는지를 6가지 방법으로 교차 검증함으로써, **"어디서 창업하느냐"보다 "어떻게 운영하느냐"가 결정적이라는 데이터 기반 의사결정 프레임워크**를 구축했습니다.

---

## 2. 프로젝트 목적

### 왜 시작했는가

독립 레스토랑의 5년 생존율은 약 50%로, 프랜차이즈(64~85%)에 비해 크게 낮습니다. 창업자와 운영자는 여전히 입지 선택을 감각에 의존하며, 기존 상권 분석 도구는 유동인구·임대료·반경 내 업체 수 정도를 제공하는 데 그칩니다.

### 어떤 문제의식이 있었는가

기존 도구에는 세 가지 공백이 있습니다:

1. **경쟁 강도가 단순 카운팅**: 맞은편 맥도날드와 골목 안쪽 서브웨이가 동일한 "경쟁자 1명"으로 집계됩니다. 두 브랜드가 독립 레스토랑에 미치는 압력의 크기는 다릅니다.
2. **소비자 반응 미반영**: 경쟁자의 품질·인지도가 높을수록 더 강한 압력입니다. 이를 정량화하는 도구가 없었습니다.
3. **생존 전략 없음**: "어디를 피해야 하는가"는 있어도, "어떻게 버텨야 하는가"에 대한 데이터 기반 답이 없었습니다.

### 왜 중요한 문제인가

경쟁 압력을 정량화하는 도구는 창업자에게는 리스크 관리 수단이, 플랫폼 기업에게는 사용자의 더 나은 의사결정을 지원하는 서비스 기능이 됩니다. 한국 외식 시장에서도 상권 선택 실패는 폐업의 주요 원인 중 하나이며, 이 방법론은 동일하게 적용 가능합니다.

### 서비스·비즈니스 관점에서 어떤 의미가 있는가

FPI는 상권 분석 서비스, 창업 플랫폼, 리뷰 기반 서비스에서 **사용자 의사결정을 지원하는 데이터 상품**으로 직결됩니다. "어느 지역의 경쟁 압력이 높은가"를 지도로 시각화하고, "어떤 전략이 그 환경에서 통하는가"를 소비자 언어로 검증한 결과는 서비스 기능 기획의 직접적인 재료가 됩니다.

## 3. 비즈니스 문제

### 해결하려고 한 문제

| 이해관계자 | 핵심 질문 |
|---|---|
| 창업 예정자 | 어느 상권에 입점하면 프랜차이즈 경쟁 압력을 덜 받을 수 있는가? |
| 기존 운영자 | 고압력 환경에서 살아남은 브랜드는 무엇이 다른가? |
| 서비스 기획자 | 경쟁 압력을 KPI로 정량화하고 사용자에게 제공할 수 있는가? |

### 프로젝트 목표

1. 소비자 행동 데이터를 기반으로 경쟁 압력을 측정하는 지수(FPI) 설계
2. FPI가 독립 브랜드의 소비자 평가(별점)와 생존에 미치는 영향을 통계적으로 검증
3. 고압력 환경에서 생존하는 브랜드와 도태되는 브랜드의 전략적 차이 규명

### 핵심 KPI

| KPI | 선정 이유 |
|---|---|
| FPI → 별점 회귀계수 유의성 (목표 p < 0.05) | 경쟁 압력이 소비자 평가에 실재하는 영향을 미치는지 검증. 비즈니스 의사결정의 전제가 되는 인과 관계 확인 |
| FPI 임계거리 (최적 반경) | "경쟁자로 볼 범위"를 데이터로 확정. 이 수치가 없으면 FPI 자체가 임의적 구성물 |
| 생존/고전 브랜드 언어 패턴 차별화 | 6가지 독립 방법론의 방향 일치 여부. 단일 방법론의 아티팩트가 아닌 실제 패턴인지 검증 |
| 소비자 감성 사전 신뢰도 (VADER 상관계수 ≥ 0.6) | FPI 구성 요소인 감성점수의 신뢰성 확보. KPI 측정 도구의 품질 검증 |

**KPI 선정 이유 요약**: 분석 결과의 신뢰도(p값, 교차 검증)가 곧 의사결정의 신뢰도이기 때문에, 통계적 엄밀성과 복수 검증을 핵심 KPI로 설정했습니다.

### 성공 기준

FPI가 통계적으로 유의미하게 별점에 영향을 미치며(p < 0.05), 생존/고전 브랜드 간에 복수 방법론이 동일하게 가리키는 해석 가능한 텍스트 패턴이 도출될 것.

Yelp Open Dataset (공개 데이터셋)

| 파일 | 전체 규모 | 분석 활용 |

### 데이터 특징

- 업체별 위치(위도·경도), 별점, 리뷰 수, 리뷰 텍스트, 영업 여부를 단일 데이터셋에서 확보
- 리뷰 수 분포가 심각하게 우편향 (중앙값 8개, 최대 7,361개) → 로그 변환 적용
- 2004~2017년 시계열 리뷰로 브랜드의 시간에 따른 변화 추적 가능

### 왜 이 데이터가 문제 해결에 적합했는가

FPI 계산에는 **위치 정보**(거리 계산), **품질 정보**(별점), **인지도 정보**(리뷰 수), **소비자 반응**(리뷰 텍스트) 네 가지가 동시에 필요합니다. Yelp 데이터는 이 네 가지를 단일 데이터셋에서 제공하는 유일한 공개 데이터셋입니다. 시계열 리뷰는 브랜드 흥망성쇠 경로를 시간 순으로 추적할 수 있어, 도태 패턴을 단면이 아닌 경로로 분석하는 데 결정적이었습니다.

## 5. 분석 흐름

### Phase 1 — 분석 가능한 환경 선택: 도시·업종 선정

**무엇을 했는가**: 3개 후보 도시(Las Vegas / Phoenix / Charlotte)를 업체 수·리뷰 수·상권 밀집도 기준으로 비교했습니다.

**왜 했는가**: FPI 분석이 통계적으로 유효하려면 프랜차이즈와 독립 브랜드가 좁은 반경 안에 충분히 혼재해야 합니다. 분석 환경 자체를 데이터로 선택해야 결과의 신뢰도가 확보됩니다.

**어떤 의사결정을 했는가**: Las Vegas 선택 (업체 5,899개, 평균 리뷰 수 158개 — Phoenix의 1.7배, Charlotte의 2.5배). Strip 중심의 고밀집 상권 구조가 거리 기반 FPI 효과를 측정하기에 자연 실험(natural experiment) 환경에 가깝다고 판단했습니다.

**서비스 관점에서 어떤 의미가 있는가**: 서비스 KPI를 측정할 때도 신호가 잘 드러나는 환경(충분한 표본, 명확한 집단 분리)을 먼저 선택해야 합니다. 데이터를 보기 전에 측정 환경을 설계하는 사고 방식입니다.

---

### Phase 2 — 측정 도구 구축: 도메인 특화 감성 사전

**무엇을 했는가**: VADER 같은 범용 감성 도구 대신, 5,261,668개 리뷰에서 음식 도메인에 특화된 감성 사전을 직접 구축했습니다.

**왜 했는가**: "crispy", "juicy", "greasy" 같은 음식 특유의 표현은 범용 사전에서 중립으로 처리됩니다. FPI의 감성 가중치가 정확해야 FPI 자체가 의미 있습니다. 측정 도구의 품질이 KPI의 품질을 결정합니다.

**어떤 의사결정을 했는가**: 별점 4~5점 리뷰(긍정 코퍼스)와 1~2점 리뷰(부정 코퍼스)에서 TF-IDF 상위 단어를 추출. 3점 리뷰는 중립 오염 방지를 위해 의도적으로 제외했습니다.

**서비스 관점에서 어떤 의미가 있는가**: 감성 사전 구축은 서비스 KPI를 위한 데이터 분류 체계(taxonomy) 구축과 동일한 작업입니다. 도메인 특성을 반영하지 않은 분류 체계는 잘못된 KPI를 생산합니다.

검증 결과: VADER 상관계수 0.724 (전체 레스토랑), 0.792 (패스트푸드) — 업종을 좁힐수록 감성 신뢰도가 높아지는 것도 확인.

---

### Phase 3 — KPI 설계: FPI 공식 구현과 문제 해결

**무엇을 했는가**: 경쟁 압력을 정량화하는 FPI 공식을 설계하고, 구현 중 발생한 음수 문제를 진단해 해결했습니다.

$$FPI_i = \sum_{j \in N(i,\,r)} \frac{stars_j \times \ln(review\_count_j+1) \times sentiment\_shifted_j}{(d_{ij}[\text{km}] + 1)^2}$$

**왜 이 공식인가 — 각 요소의 설계 근거**:
- `stars_j`: 품질이 높은 프랜차이즈일수록 소비자 기대치를 더 높게 올린다
- `ln(review_count+1)`: 리뷰 수가 인지도를 반영하지만 극단값이 크므로 로그 스케일 적용
- `sentiment_shifted`: 별점과 텍스트 감성이 다를 수 있으므로 소비자 체감 품질 반영
- `(d+1)^{-2}`: 가까울수록 경쟁 압력이 강하게 작용하는 거리 역제곱 감쇠

**의사결정 — 음수 문제 해결**: 감성점수 음수값(전체 72~74%)이 FPI를 음수로 만드는 구조적 문제를 발견했습니다. MinMax 정규화(상대 간격 손실)와 상수 추가(임의적) 두 대안을 검토한 후, **Shift 정규화**(최솟값 차감)를 채택했습니다. 음수 제거와 상대적 간격 보존을 동시에 달성하는 유일한 방법이었기 때문입니다.

**서비스 관점에서 어떤 의미가 있는가**: KPI 설계는 개념 정의 → 변수 분해 → 수식화 → 문제 해결의 반복 과정입니다. "경쟁 압력"이라는 추상 개념을 측정 가능한 수식으로 만든 경험은, 서비스에서 "사용자 만족도"·"참여도"·"이탈 위험"을 정량화하는 KPI 설계 과정과 동일한 구조입니다.

### Phase 4 — 파라미터를 데이터로 확정: 임계거리 결정

**무엇을 했는가**: "몇 미터 반경 내 프랜차이즈를 경쟁자로 볼 것인가"를 임의로 설정하지 않고, 100m/200m/300m/500m 네 가지 반경을 모두 검증해 데이터로 결정했습니다.

**왜 했는가**: KPI의 파라미터를 임의로 설정하면 결과를 신뢰할 수 없습니다. "왜 이 수치인가"에 명확한 근거가 있어야 이해관계자에게 설명할 수 있습니다.

**의사결정 결과**: 300m 확정. 별점과 감성점수 두 종속변수 모두 p < 0.05를 만족하는 최소 반경. PART 1과 PART 2에서 독립적으로 동일한 결과가 나왔습니다.

**서비스 관점에서 어떤 의미가 있는가**: 300m는 보행 5분 이내 거리로, 소비자가 식사 장소를 결정할 때 자연스럽게 비교하는 반경과 일치합니다. 서비스에서도 KPI의 기준값을 사용자 행동 데이터로 검증해야 한다는 원칙과 같습니다.

### Phase 5 — 분석 중 한계 발견과 구조적 해결: PART 2 설계

**무엇을 했는가**: PART 1 텍스트 분석 중 두 가지 구조적 한계를 발견하고, 데이터 근거를 제시해 팀의 방향을 전환했으며, PART 2 전체를 독립적으로 설계·수행했습니다.

**발견한 문제**:
1. **업종 이질성**: 태국 요리·스시·버거·피자가 혼재해 FPI 구간별 키워드가 "경쟁 환경 차이"가 아닌 "업종 차이"를 반영 (PCA Biplot: PC1 6.3%, PC2 4.7% — 업종 이질성이 경쟁 효과보다 분산을 더 크게 설명)
2. **카지노 노이즈**: 생존/고전 브랜드 비교에서 `hotel, casino, MGM` 키워드가 상위를 점령해 음식·서비스 품질 차이 분석 불가

**의사결정**: Burgers + Sandwiches + Fast Food 3개 카테고리로 범위를 좁히는 PART 2를 설계했습니다. Pizza·Chicken은 배달 중심·가족 단위 수요로 소비 구조가 달라 제외했습니다.

**결과**: 업종 통제 후 FPI 효과가 2.3배 강해졌습니다 (β = −0.012 → β = −0.027).

**서비스 관점에서 어떤 의미가 있는가**: 분석 결과가 예상과 다를 때 "데이터 문제인가, 방법론 문제인가, 환경 문제인가"를 진단하는 능력이 핵심입니다. 이 단계는 분석의 노이즈를 식별하고 제거하는 **서비스 QA 과정**과 동일한 사고 구조입니다.

### Phase 6 — 예상치 못한 결과의 재해석: 폐업 예측 변수 규명

**무엇을 했는가**: FPI가 폐업 여부에 직접적인 영향을 미치지 않는다는 결과(p = 0.401)를 분석 실패로 처리하지 않고, 실제 폐업을 결정하는 변수가 무엇인지 탐색했습니다.

**의사결정 및 발견**: 폐업 여부를 가장 강하게 예측하는 변수는 **소비자 참여도(리뷰 수, OR = 1.621, p < 0.001)**임을 발견했습니다. 리뷰 수가 1% 증가할 때 영업 중 확률이 약 62% 높아집니다.

**데이터 구조적 한계**: 횡단면 데이터(2017년 스냅샷)로 인해 FPI → 별점 → 폐업의 완전한 매개 경로는 통계적으로 성립하지 않았습니다. 폐업 브랜드 평균 별점(3.60)이 영업 중 브랜드(3.52)보다 오히려 높은 역설이 발생했으며, 이는 데이터 수집 시점의 구조적 한계에서 비롯됩니다. [확인 필요: Bootstrap 매개효과 분석 실시 여부 및 결과]

**서비스 관점에서 어떤 의미가 있는가**: 경쟁 압력이 높다고 곧바로 폐업하는 것이 아닙니다. 폐업을 직접 결정하는 것은 소비자와의 지속적 관계(리뷰 참여)입니다. 이는 "좋은 입지"보다 "꾸준한 고객 관계 형성"이 더 중요하다는 처방적 인사이트로, 서비스에서 User Acquisition보다 Retention이 장기 생존을 결정하는 원리와 동일합니다.

### Phase 7 — 생존 전략 규명: 6가지 방법론 교차 검증

**무엇을 했는가**: HP(고압력) 구간 내 생존 브랜드(별점 ≥ 4.0, 63개)와 고전 브랜드(별점 ≤ 3.0, 65개)를 6가지 독립적인 텍스트 분석 방법으로 비교했습니다.

**왜 6가지를 사용했는가**: 단일 방법론의 결과는 그 방법론 고유의 아티팩트일 수 있습니다. TF-IDF 차이 / N-gram / VADER 감성 강도 / LDA 토픽 모델링 / 프랜차이즈 직접 언급 / 시계열 감성 변화 — 6가지 독립 방법이 동일한 방향을 가리킬 때 비로소 "우연이 아닌 패턴"이라고 주장할 수 있습니다.

**서비스 관점에서 어떤 의미가 있는가**: 서비스 의사결정에서도 단일 지표(예: DAU)만 보는 것은 위험합니다. 복수의 지표(리텐션, NPS, 이탈률)가 동일한 방향을 가리킬 때 비로소 서비스 개선의 근거가 됩니다.

## 6. 주요 분석 결과

### 결과 1 (최우선) — 경쟁 압력은 소비자 평가에 실재하는 음(−)의 영향을 미친다

**결과**: FPI 1단위 증가 시 독립 브랜드 별점 0.012점 하락(PART 1, p = 0.033). 패스트푸드 업종으로 좁히면 0.027점 하락으로 효과가 2.3배 강해짐(PART 2, p = 0.048).

| 분석 | FPI 계수 | p-value | 별점 설명력 |
|---|---|---|---|
| PART 1 (전체 Restaurants) | −0.0119 | 0.033 ✅ | R² = 0.072 |
| PART 2 (패스트푸드) | **−0.0271** | 0.048 ✅ | R² = 0.083 |

**근거**: OLS + HC3 강건 표준오차 적용 다중회귀분석. 상권 특성(16개 동네 더미)과 매장 규모(로그 리뷰 수)를 통제한 순수 FPI 효과.

**의미**: "프랜차이즈 경쟁 압력"이 소비자 평가에 미치는 영향을 처음으로 정량화했습니다. 업종 이질성을 통제할수록 신호가 더 선명해진다는 것도 확인했습니다.

**서비스/비즈니스 활용 가능성**: 상권 분석 서비스에서 "이 위치에서 창업 시 경쟁 압력으로 인한 예상 별점 하락폭"을 사전에 제시하는 기능에 직접 활용 가능합니다.

---

### 결과 2 — 폐업을 결정하는 것은 경쟁 압력이 아닌 소비자 참여도다

**결과**: 로지스틱 회귀에서 FPI는 폐업에 직접 영향 없음(p = 0.401). 반면 리뷰 수(OR = 1.621, p < 0.001)가 가장 강력한 생존 예측 변수.

| 변수 | Odds Ratio | p-value |
|---|---|---|
| FPI_300m | 0.966 | 0.401 ❌ |
| 별점(stars) | 0.741 | 0.005 |
| log(리뷰수) | **1.621** | **< 0.001 ✅** |

**의미**: 입지가 나쁘다고 곧바로 폐업하는 것이 아닙니다. 고압력 환경에서도 소비자와 지속적으로 소통하며 리뷰를 쌓는 브랜드는 생존합니다. 경쟁 대응 전략의 우선순위를 "입지 회피"보다 "고객 관계 구축"에 둬야 한다는 근거입니다.

**서비스/비즈니스 활용 가능성**: 리뷰 플랫폼에서 리뷰 활동량 관리를 생존 전략의 핵심 기능으로 포지셔닝할 수 있습니다.

---

### 결과 3 — 생존 브랜드는 6가지 방법이 동일하게 가리키는 차별화 패턴을 가진다

동일 HP 상권 내 생존 브랜드 평균 VADER 감성 강도 0.738 vs 고전 브랜드 0.454 (Mann-Whitney p < 0.001). 62% 더 강한 긍정 반응.

| 분석 방법 | 생존 브랜드 특성 | 고전 브랜드 특성 |
|---|---|---|
| TF-IDF 차이 | delicious, amazing, flavor, perfect | bad, never, wait |
| N-gram | crab leg, truffle parmesan fry, chicken waffle | hot dog, white castle, wait minute |
| VADER | 평균 0.738 (하위 25% = 0.778) | 평균 0.454 (하위 25% = 중립) |
| LDA 토픽 | 시그니처 메뉴 + 서비스 경험 토픽 | 저가 메뉴 + 서비스 불만 토픽 |
| 프랜차이즈 언급 | homemade 90.6회/만, house made 29.4회/만 | corporate 61.1회/만, fast food chain 33.1회/만 |
| 시계열 | 완만한 하락 (기울기 −0.078) | 급격한 하락 (기울기 −0.300) |

**의미**: 생존과 도태를 가르는 것은 운이나 입지가 아닙니다. 소비자가 자발적으로 "homemade", "house made", "family owned"를 사용한다는 것은 독립 브랜드로서의 정체성을 소비한다는 의미입니다.

**서비스/비즈니스 활용 가능성**: 리뷰 감성 시계열 모니터링으로 브랜드 품질 저하를 조기 감지하는 알림 서비스, 생존 브랜드 키워드 패턴을 기반으로 마케팅 언어를 가이드하는 기능 기획에 활용 가능.

---

### 결과 4 — 고전 브랜드는 처음부터 나쁘지 않았다: "한때는 좋았는데" 패턴

**결과**: 시계열 분석에서 고전 브랜드가 2006~2008년 일부 시기에 생존 브랜드와 비슷한 감성 수준을 기록했습니다.

초기(2005~2008) 주요 키워드: buffet, crab leg, prime rib → 차별화 메뉴  
후기(2014~2017) 주요 키워드: white castle, employee, staff, frozen → 저가화 + 직원 문제

도태 경로 재구성:
```
초기 차별화 메뉴 (뷔페·델리·해산물)
→ 소비자 유입·리뷰 급증
→ 품질 유지 실패 (직원 불만 키워드 등장)
→ 저가 메뉴로 전환 (white castle, frozen)
→ "McDonald's보다 못하다" (소비자 최소 기준선 미달)
→ 별점 3.0 이하 고착화
```

실제 리뷰: *"This place has gone downhill from years ago. In the past it was really awesome... now it's cafeteria style."*

**의미**: 창업 초기 성공이 지속 가능성을 보장하지 않습니다. 규모가 커질수록 품질 유지 실패가 더 치명적입니다. 스타트업의 PMF 이후 단계와 동일한 문제 구조입니다.

**서비스/비즈니스 활용 가능성**: "초기 고평가 이후 감성점수 하락 추세 감지" 조기 경보 기능 — 브랜드가 도태 경로에 진입했을 때 선제적 알림 제공.

---

### 결과 5 — McDonald's가 소비자의 암묵적 최소 기준선으로 작동한다

생존 브랜드 리뷰: *"way better than McDonald's"*, *"blows McDonald's out of the water"*  
고전 브랜드 리뷰: *"McDonald's would have been better"*, *"reminded me of plain McDonald's"*

**의미**: 소비자는 McDonald's를 패스트푸드 업종의 최소 기준으로 암묵적으로 설정합니다. 독립 브랜드는 이 기준을 "어떤 차원에서든" 명확히 초과해야 존재 이유가 생깁니다. 경쟁 상대는 인근 프랜차이즈만이 아니라 소비자의 머릿속 기준이기도 합니다.

**서비스/비즈니스 활용 가능성**: 사용자의 "암묵적 비교 대상"을 리뷰·언급 데이터로 추출하는 방법론은 서비스 포지셔닝 전략에도 동일하게 적용됩니다.

## 7. 핵심 인사이트

### 인사이트 1: 입지보다 전략이 결정적이다 → "어떻게 운영할 것인가"를 먼저 설계하라

동일한 HP 상권 안에서 별점 4.0 이상 생존 브랜드와 별점 3.0 이하 고전 브랜드가 뒤섞여 있습니다. 위치 자체가 운명을 결정하지 않습니다.

> 입지 선택은 필요조건이지 충분조건이 아닙니다. 어디서 창업하느냐보다 어떤 정체성으로, 어떤 수준의 품질을 얼마나 오래 유지하느냐가 생존을 결정합니다.

전략적 함의: 상권 분석 서비스는 "좋은 입지 찾기"에서 "입지별 맞춤 생존 전략 제시"로 기능을 확장해야 합니다.

---

### 인사이트 2: 경쟁 압력은 즉각적 충격이 아닌 장기적 소진으로 작동한다 → 대응 전략의 시간 지평을 중장기로 설계하라

FPI가 폐업에 직접 영향을 미치지 않는다는 발견은 더 정확한 현실을 보여줍니다. 경쟁 압력은 서서히 별점을 갉아내고, 결국 소비자 참여가 줄어들면서 폐업에 이릅니다.

> 단기 매출보다 장기 소비자 관계(리뷰 수, 감성 추세)를 모니터링해야 합니다.

---

### 인사이트 3: 생존 브랜드는 "프랜차이즈가 줄 수 없는 것"을 소비자 언어로 증명한다

homemade·house made·family owned 언급이 고전 브랜드 대비 3~10배 높고, 복합 시그니처 표현("truffle parmesan fry", "oxtail chili cheese")이 소비자 리뷰에 자발적으로 등장합니다.

> 독립 브랜드의 경쟁 우위는 "프랜차이즈와 다르다"는 것을 소비자가 직접 언어화하도록 만드는 정체성 전략에서 나옵니다.

서비스 기획에서의 함의: 사용자가 "이 서비스는 XX와 달리 ..."라고 자발적으로 표현할 때 브랜드 차별화가 성공한 것입니다. 이 언어를 리뷰 데이터에서 추출해 기능 우선순위에 반영하는 것이 데이터 기반 서비스 전략입니다.

---

### 인사이트 4: 도태는 초기 역량 부족이 아닌 "성장 이후 품질 유지 실패"에서 시작된다

고전 브랜드가 초기에 높은 감성 점수를 기록했다는 발견은 "처음부터 문제가 있었다"가 아니라 "성공한 이후가 더 어렵다"는 것을 보여줍니다.

> 창업 전략("어떻게 시작할 것인가")만큼 스케일업 전략("성장 이후 품질을 어떻게 유지할 것인가")이 중요합니다.

## 8. 서비스 개선 전략 및 비즈니스 가치

### 우선순위 1: 입지 리스크 평가 도구

**전략**: FPI 지도를 창업 전 상권 리스크 수치화 도구로 제공  
**근거**: NP 구간 평균 별점 3.83 vs HP 구간 3.61 — FPI 수치화로 특정 위치의 경쟁 압력 사전 확인 가능  
**실행 방안**: FPI 인터랙티브 지도 → 업종 필터·반경 조절 → 입지별 예상 경쟁 압력 점수 제공

### 우선순위 2: 생존 전략 가이드라인 (운영자 대상)

**전략**: 고압력 환경 창업자에게 생존 브랜드 언어 패턴 기반 마케팅 가이드 제공

생존 공식: **시그니처 메뉴 × 수제·오너십 정체성 × 품질 지속성**

1. McDonald's 기준선을 맛·신선함·경험 중 하나의 차원에서 소비자가 인지할 수 있게 초과하라
2. 소비자가 메뉴 조합 전체를 언어로 기억하게 하라 ("truffle parmesan fry", "oxtail chili cheese")
3. homemade·house made·family owned 언어를 적극 활용해 정체성 차별화를 유도하라

### 우선순위 3: 브랜드 건강 모니터링 (경보 시스템)

**전략**: 리뷰 감성 시계열 추세 모니터링으로 도태 경로 조기 감지  
**근거**: 고전 브랜드는 초기 높은 감성 점수에서 급격히 하락(기울기 −0.300) — 하락 시작 시점에 개입 가능  
**실행 방안**: 연도별 감성 점수 추세 알림, "직원·관리 불만 키워드 급증" 조기 경보, 저가 메뉴 전환 패턴 감지

### 우선순위 4: 데이터 상품화 / API 제공

**전략**: FPI 계산 로직을 API로 제공해 상권 분석 서비스·창업 플랫폼에 B2B 연동  
**비즈니스 가치**: 프랜차이즈 본사 신규 출점 전략, 독립 창업자 입지 비교, 상권 컨설팅 자동화

3. **텍스트 분석 방법론 확장**: PART 1의 TF-IDF 차이 한 가지를 PART 2에서 6가지 독립 방법으로 확장했습니다.
4. **임계거리 민감도 분석**: 100m/200m/300m/500m 반경 전수 검증으로 300m를 데이터 기반 확정.

### 주요 의사결정

- PART 1 → PART 2 방향 전환 제안 (PCA 결과 + 카지노 노이즈를 데이터 근거와 함께 팀에 제시)
- PART 2 업종 범위 확정 (Pizza·Chicken 제외 근거 수립)
- 감성 사전 구축 시 3점 리뷰 의도적 제외 결정 (중립 오염 방지)

### 협업 내용

PART 1에서 FPI 산출(04) 및 회귀분석(05) 노트북은 팀원이 수행했으며, 음수 문제 발견 및 보완, 결과 해석 검토를 담당했습니다.

### 기여도

PART 2 6개 노트북 완전 독립 수행, PART 1 핵심 분석(텍스트·지도·보완) 주도. [확인 필요: 전체 노트북 수 및 PART 1 담당 노트북 수]

## 11. 포트폴리오에서 강조해야 할 차별점

### 차별점 1: 데이터에 없는 KPI를 처음부터 설계하고 두 번 검증했다

"프랜차이즈 경쟁 압력"은 어떤 데이터셋에도 존재하지 않는 변수입니다. 개념 정의 → 변수 분해 → 수식화 → 검증의 전 과정을 완결한 것은 **KPI 설계 전 과정의 경험**입니다. 실무에서 "측정하고 싶은데 데이터에 없다"는 상황에서 가장 필요한 능력입니다.

### 차별점 2: 모든 파라미터를 "왜 이 수치인가"로 설명할 수 있다

프랜차이즈 판정 임계값(민감도 분석 → 10회), 임계거리(4개 반경 검증 → 300m), 감성 사전(교집합 제거 방식) — 모든 수치에 데이터 근거가 있습니다. 이해관계자에게 "왜 이 수치인가"를 설명할 수 있는 분석은 신뢰도가 다릅니다.

### 차별점 3: 예상치 못한 결과에서 더 깊은 인사이트를 도출했다

FPI가 폐업에 직접 영향을 미치지 않는다는 결과를 실패로 처리하지 않고, 실제 폐업 예측 변수(리뷰 수, OR = 1.621)를 발견했습니다. 데이터가 "아니오"라고 말할 때 "왜 아니오인가"를 파고들 수 있음을 보여줍니다.

### 차별점 4: 분석 한계를 회피하지 않고 구조적으로 해결했다

PART 1에서 업종 이질성을 발견했을 때, 그 데이터로 결론을 내리거나 한계를 주석으로 얼버무리지 않았습니다. 원인을 정확히 진단하고 PART 2 전체를 재설계해 FPI 효과를 2.3배 강화했습니다.

### 차별점 5: 동일 결론을 6가지 독립 방법으로 교차 검증했다

TF-IDF / N-gram / VADER / LDA / 시계열 / 직접 언급 — 6가지가 동일한 방향을 가리킵니다. 하나의 방법론에 의존한 결론이 아닌 **복수의 독립 증거가 수렴하는 결론**입니다.

### 왜 다른 지원자의 Yelp 프로젝트보다 경쟁력이 있는가

대부분의 Yelp 프로젝트는 별점 예측 모델이나 리뷰 감성 분류에 집중합니다. 이 프로젝트는 **비즈니스 질문("왜 어떤 브랜드는 살아남고 어떤 브랜드는 사라지는가")을 먼저 설정하고, 이를 위한 KPI를 직접 설계하고, 부정 결과까지 의미 있는 인사이트(리뷰 수의 결정적 역할)로 전환했습니다.** 기술이 아닌 문제 해결 과정이 주인공인 프로젝트입니다.

## 12. 채용 담당자 관점에서 가장 경쟁력 있는 경험 TOP 3

### 경험 1: 존재하지 않는 지표를 설계하고 두 번 검증했다

> FPI 개념 정의 → 공식 설계 → 음수 문제 발견·해결 → PART 1 검증(β = −0.012, p = 0.033) → PART 2 독립 재검증(β = −0.027, p = 0.048)

**왜 중요한가**: 실무에서 가장 가치 있는 분석가는 "측정되어야 하는데 아직 측정되지 않는 것"을 정의하고 측정하는 사람입니다. 이 경험은 그 능력을 구체적인 공식과 두 번의 독립 검증으로 보여줍니다.

**보여주는 역량**:
- 문제 정의: "경쟁 압력이란 무엇인가"를 측정 가능한 변수로 분해
- KPI 설계: 개념 → 공식 → 파라미터 확정의 전 과정
- 비즈니스 이해: 소비자 행동(거리 감쇠, 인지도 가중치)을 공식에 반영
- 실행력: 구현 중 문제 발견 → 해결까지 스스로 완결

---

### 경험 2: 분석 중 한계를 스스로 발견하고 PART 2 전체를 재설계했다

> PART 1 텍스트 분석 중 업종 이질성 + 카지노 노이즈 발견 → 데이터 근거 제시 → PART 2 독립 설계 및 수행 → FPI 효과 2.3배 강화

**왜 중요한가**: 주어진 분석을 완성하는 것과, 분석이 잘못된 방향으로 가고 있음을 발견하는 것은 완전히 다른 능력입니다. 이 경험은 구조적 문제를 정확히 진단하고, 팀에 근거를 제시해 방향을 바꾸며, 새로운 분석을 독립 수행한 자기 주도성과 협업 능력을 동시에 보여줍니다.

**보여주는 역량**:
- 문제 정의: PART 1의 노이즈 발생 원인을 정확히 진단
- 협업: PCA 결과 + 카지노 키워드를 데이터 근거로 제시해 방향 전환 설득
- 실행력: PART 2 6개 분석 독립 설계 및 수행

---

### 경험 3: 예상치 못한 결과에서 더 풍부한 인사이트를 만들었다

> FPI → 폐업 직접 효과 없음(p = 0.401) 발견 → "왜 없는가" 탐색 → 리뷰 수(OR = 1.621)가 핵심 변수임 발견

**왜 중요한가**: "가설이 기각되었을 때" 어떻게 대응하는가가 분석가의 수준을 결정합니다. 이 경험은 "FPI가 폐업을 직접 결정하지 않는다면, 무엇이 결정하는가"라는 더 중요한 질문을 찾아내 답까지 도출했습니다.

**보여주는 역량**:
- 문제 정의: "직접 효과 없음"을 "그렇다면 진짜 결정 요인은 무엇인가"로 전환
- 데이터 분석: Logistic 회귀에서 OR = 1.621 발견 및 해석
- 비즈니스 이해: "경쟁 압력보다 고객 관계가 생존을 결정한다"는 비직관적 결론의 현실적 의미 도출
- 스토리텔링: 부정 결과를 더 풍부한 인사이트로 전환

### AI 서비스 검토

FPI는 소비자 데이터(리뷰, 별점, 위치)를 구조화하여 새로운 서비스 가치를 창출하는 프로세스입니다. AI 기반 상권 분석·추천 서비스에서 어떤 데이터를 어떤 방식으로 구조화해야 하는지에 대한 직접적인 경험입니다. "AI가 어떤 피처를 학습해야 하는가"를 먼저 설계하는 관점이 이 프로젝트에 내재되어 있습니다.

### 기능 기획

생존/고전 브랜드 분석 결과는 서비스 기능으로 직결됩니다:
- **입지 리스크 점수 표시 기능**: FPI 수치화 → 특정 위치의 경쟁 압력 수치 제공
- **브랜드 건강 감성 추세 모니터링**: 시계열 감성 분석 → 도태 경로 조기 감지 알림
- **생존 전략 가이드**: homemade·시그니처 메뉴 키워드 기반 마케팅 언어 추천

분석 결과가 어떤 기능으로 전환될 수 있는지를 사용자 시나리오 단위로 생각하는 능력이 이 포트폴리오에서 확인됩니다.

### KPI 설계

FPI는 기존에 없던 KPI를 처음부터 설계한 경험입니다:
- **개념 정의**: "경쟁 압력이란 무엇인가"
- **변수 분해**: 품질·인지도·감성·거리
- **파라미터 검증**: 임계거리 4개 반경 민감도 분석
- **신뢰도 검증**: VADER 교차 검증, 두 분석 독립 재현

이 프레임워크는 서비스 KPI(사용자 만족도, 참여도, 이탈 위험) 설계에 직접 적용 가능합니다.

### 데이터 분류체계 구축

Yelp 데이터에 프랜차이즈/독립 브랜드 레이블이 없는 상황에서 브랜드명 정제 → 임계값 민감도 분석 → 화이트리스트 수동 추가라는 체계적 분류 방법론을 설계했습니다. 비정형 데이터에서 분류 체계를 구축하는 경험으로, 서비스 데이터 정제·분류 업무에 직결됩니다.

### 서비스 QA

PART 1에서 업종 이질성과 카지노 노이즈를 발견해 PART 2로 방향을 전환한 과정은, 분석 결과의 품질을 검증하고 문제를 발견하는 서비스 QA 프로세스와 동일한 구조입니다. 결과가 예상과 다를 때 "데이터 문제인가, 방법론 문제인가, 환경 문제인가"를 진단하는 능력입니다.

### 사업 기획 지원

FPI 분석 결과는 세 종류의 이해관계자(창업자·운영자·상권 분석 서비스)에게 각각 다른 가치를 제공합니다. 하나의 분석에서 복수 이해관계자의 의사결정을 지원하는 방식은 미래전략실의 사업 기획 지원 역할과 동일한 구조입니다.

### 문서화

각 의사결정의 근거(왜 이 도시인가, 왜 이 반경인가, 왜 이 공식인가)를 명확히 기록한 분석 보고서는, 미래전략실에서 요구되는 "근거 있는 설명 능력"과 의사결정 문서 작성 역량을 보여줍니다.

### 현직 PM/데이터 전략 담당자 관점에서 보완하면 좋을 점

**1. 숫자를 의사결정으로 연결하는 문장을 추가하라**

"β = −0.027"이라는 통계 수치보다 "FPI가 1단위 높은 상권에서 창업하면 별점이 평균 0.027점 낮다. HP 구간 평균 별점 3.61 기준에서 이 수치는 3.5점 이하 등급 진입을 의미하며, 검색 노출 감소로 이어질 수 있다"처럼 **숫자 → 사용자/운영자의 구체적 행동 변화**로 연결하는 문장이 필요합니다.

**2. 서비스 기획 시나리오를 사용자 관점으로 재서술하라**

"FPI 지도 제공"이라는 기능 서술보다 "창업 예정자가 원하는 지역의 FPI 점수를 확인하고, 예상 별점 하락폭을 계산해 입지 비교표를 만드는 시나리오"처럼 **누가 어떻게 사용하는가**로 구체화하면 서비스 기획 역량이 더 선명하게 드러납니다.

**3. 한계를 "다음 프로젝트"로 전환하라**

R² = 0.08, 단일 도시 한계를 "아쉬운 점"이 아닌 "ABSA로 음식·서비스·가격 차원을 분리하면 생존 요인을 더 정밀하게 볼 수 있다", "Phoenix·Charlotte를 비교군으로 추가하면 라스베가스 결과의 일반화 가능성을 검증할 수 있다"처럼 **발전 방향**으로 제시하면 성장 잠재력으로 읽힙니다.

**4. PART 1 → PART 2 전환을 "의도된 설계"로 재프레이밍하라**
