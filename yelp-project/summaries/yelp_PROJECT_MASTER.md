## 1. 프로젝트 개요

### 프로젝트명
**Yelp 데이터 기반 프랜차이즈 경쟁압력 지수(FPI) 설계 및 독립 레스토랑 생존 전략 분석**

### 진행 기간
확인 필요 (노트북 커밋 이력 기준으로 추후 보완)

### 팀 구성
- 팀 프로젝트 (규모: 확인 필요)
- 본인 역할:
  - **PART1**: 데이터 준비(01), 텍스트 분석(06), 생존 브랜드 분석(07), FPI 지도(08) 전담 + 회귀분석(05), FPI 산출(04) 보완
  - **PART2**: 전체 6개 노트북 전담 (업종 특화 심화 분석 독립 수행)

### 프로젝트 배경
Yelp 공개 데이터셋을 활용한 비즈니스 분석 프로젝트. 독립 레스토랑이 주변 프랜차이즈로부터 받는 경쟁 압력을 어떻게 정량화할 수 있는가, 그리고 이 압력이 독립 브랜드의 성과와 생존에 어떤 영향을 미치는가라는 문제의식에서 출발했다.

Las Vegas는 미국 내 Yelp 데이터 기준 업체 수(5,899개)·리뷰 수(1,603,616개)·평균 리뷰 수(158개)가 모두 가장 풍부한 도시로, 관광 중심 상권이라는 특수성이 있어 프랜차이즈-독립 브랜드 간 경쟁 구조를 분석하기에 적합했다.

### 해결하고자 한 비즈니스 문제

| 관점 | 문제 |
|---|---|
| 창업자 | "어느 상권에 입점하면 프랜차이즈 경쟁 압력을 덜 받을 수 있는가?" |
| 독립 브랜드 운영자 | "고압력 환경에서 살아남은 브랜드는 무엇이 다른가?" |

| 연구자 | "경쟁 압력을 단일 지수로 정량화하는 것이 통계적으로 유효한가?" |

## 4. 분석 과정 상세 정리

### 4-1. 프랜차이즈 판정 기준 설계

**무엇을 했는가**: 동일 브랜드명이 몇 개 이상 등장해야 프랜차이즈로 판정할지 임계값을 설계했다.

**왜 했는가**: "프랜차이즈"의 공식 정의는 없고, 데이터에 프랜차이즈 여부를 나타내는 컬럼이 없었기 때문에 분석팀이 직접 판정 기준을 만들어야 했다.

**대안과 선택**:

| 임계값 | 프랜차이즈 비율 | 선택 여부 |
|---|---|---|
| 3회 | 27.1% | ❌ (지역 체인까지 포함, 과다 분류) |
| 5회 | 21.7% | ❌ |
| **10회** | **16.7%** | **✅ 채택** |
| 20회 | 12.6% | ❌ (전국 대형 체인만 포함, 과소 분류) |

→ 10회 기준 + 화이트리스트(Subway, Chipotle 등 전국 체인 수동 추가) 2단계 방식 채택.  
→ PART2에서는 패스트푸드 특성상 화이트리스트 35개로 확대.

⭐ **채용 관점**: 기준이 없는 상황에서 민감도 분석으로 임계값을 스스로 결정한 것. "데이터에 없는 개념을 어떻게 측정했는가"에 대한 명확한 근거 제시가 가능하다.

### 4-2. TF-IDF 기반 감성 사전 자체 구축

**무엇을 했는가**: VADER 같은 기성 도구를 쓰지 않고, 별점을 정답지(ground truth)로 활용해 음식 리뷰에 특화된 감성 사전을 직접 만들었다.

**왜 했는가**: 기성 감성 분석 도구는 범용 목적으로 훈련되어 있어 음식 도메인의 "crispy", "juicy" 같은 단어를 정확히 반영하지 못한다. 도메인 특화 사전이 더 높은 정확도를 낼 것이라는 가설을 세웠다.

**설계 방식**:
1. 별점 4~5점 리뷰 → 긍정 코퍼스, 1~2점 → 부정 코퍼스 (3점은 의도적 제외 — 중립 오염 방지)
2. 각 코퍼스에서 TF-IDF 상위 300개 단어 추출
3. 긍정/부정 교집합(PART1: 216개, PART2: 204개) 제거 → 순수 사전 확보
4. 감성점수 = (긍정 단어 수 - 부정 단어 수) / 전체 단어 수

**검증 방법**: 자체 감성점수와 VADER를 비교 (Pearson 상관)
- PART1: 상관계수 0.724 (p < 0.001) ✅
- PART2: 상관계수 0.792 (p < 0.001) ✅ — 업종 특화로 신뢰도 향상

⭐ **채용 관점**: "왜 기존 도구를 그냥 쓰지 않았는가"에 답할 수 있는 의사결정 과정. 검증까지 수행해 신뢰도를 확보한 것이 핵심.

### 4-3. FPI 공식 설계

**무엇을 했는가**: 독립 브랜드가 주변 프랜차이즈로부터 받는 경쟁 압력을 단일 지수(FPI)로 정량화하는 공식을 설계했다.

**FPI 공식**:

$$FPI_i = \sum_{j \in N(i,r)} \frac{\text{stars}_j \times \ln(\text{review\_count}_j+1) \times \text{sentiment\_shifted}_j}{(d_{ij}[\text{km}] + k)^2}$$

**각 요소의 설계 의도**:

| 구성 요소 | 역할 | 설계 근거 |
|---|---|---|
| `stars` | 프랜차이즈 품질 가중치 | 별점이 낮은 프랜차이즈는 경쟁압력이 낮다고 가정 |
| `ln(review_count+1)` | 영향력 규모 | 리뷰가 많을수록 인지도가 높아 경쟁 압력 크다고 가정. log 변환으로 극단값 완화 |
| `sentiment_shifted` | 소비자 반응 | 감성이 좋은 프랜차이즈일수록 경쟁 압력 크다고 가정 |
| `(거리 + k)²` | 거리 감쇠 | 가까울수록 영향 강함. k=1(km)로 발산 방지 |

**Shift 정규화 추가 (수정 사항)**:  
초기 설계에서 `tfidf_sentiment`를 그대로 사용했더니 음수 감성점수를 가진 프랜차이즈(PART1: 72.3%, PART2: 73.9%)로 인해 FPI 자체가 음수가 되는 문제가 발생했다.

→ MinMax 정규화는 절대 크기를 잃고, 상수 추가는 임의성이 강하여 **Shift 정규화**(최솟값만 차감)를 채택. 음수 제거 + 상대적 간격 보존 + 절대 크기 유지.

⭐ **채용 관점**: FPI 자체가 팀의 오리지널 설계물. "지표를 직접 설계해본 경험"을 구체적 공식과 함께 설명할 수 있다. 문제 발견(음수) → 대안 비교 → 해결(Shift) 과정이 완결성을 갖춘다.

### 4-4. 임계거리 300m 확정 (민감도 분석)

**무엇을 했는가**: FPI 계산 시 "몇 미터 반경 내의 프랜차이즈를 포함할 것인가"를 데이터 기반으로 결정했다.

**방법**: 반경 100m/200m/300m/500m 각각 FPI를 산출한 뒤, 각 FPI가 별점 및 감성점수와 유의미한 관계(p<0.05)를 보이는지 OLS 단순회귀로 검증.

**결과 (PART1)**:

| 반경 | 별점 p | 감성점수 p | 채택 |
|---|---|---|---|
| 100m | 0.0001 ✅ | 0.2672 ❌ | — |
| 200m | 0.0002 ✅ | 0.5950 ❌ | — |
| **300m** | **0.0000 ✅** | **0.0207 ✅** | **채택** |
| 500m | 0.0000 ✅ | 0.0017 ✅ | — |

→ 두 종속변수 모두 유의미한 최소 반경 = 300m.  
→ PART2에서 재검증해도 300m 동일하게 확정됨.

⭐ **채용 관점**: 임의로 반경을 설정하지 않고 데이터로 검증했다는 점. "왜 300m인가"에 명확하게 답할 수 있다.

### 4-6. PART1 → PART2 방향 전환 결정

**전환 배경**:  
PART1 텍스트 분석(06, 07) 수행 중 두 가지 구조적 문제를 발견했다.

1. **업종 이질성**: 태국음식·스시·버거·피자가 같은 데이터셋에 있어 FPI 구간별 키워드가 "경쟁 환경의 차이"가 아닌 "업종 차이"를 반영하고 있었다.

**전환 결정**: 동일한 소비 및 경쟁 메커니즘을 공유하는 업종으로 범위를 좁혀 재분석. `Burgers + Sandwiches + Fast Food` 3개 카테고리 선택.  
- Pizza, Chicken은 배달 중심·저녁 수요 비중이 높아 입지 경쟁 구조가 다르다고 판단하여 제외.

⭐ **채용 관점**: 분석 중 한계를 스스로 발견하고 방향을 전환한 것. "분석이 이상하다"고 느낀 순간을 정확히 진단하고 해결한 문제해결 능력의 증거.

### 4-7. PART2 매개효과 분석 시도 및 한계

**무엇을 했는가**: FPI가 폐업에 직접 영향을 미치는지, 별점 또는 리뷰수를 통해 간접 영향을 미치는지 두 가지 매개 경로를 검토했다.

**설계 배경**: Logistic 회귀에서 FPI 직접 효과(p=0.401)가 유의미하지 않았다. 이에 FPI → 별점 → 폐업, FPI → 리뷰수 → 폐업 두 경로의 성립 여부를 검토했다.

**결과**:

| 매개 경로 | 경로 a | 경로 b | 성립 여부 |
|---|---|---|---|
| FPI → 별점 → 폐업 | p=0.048 ✅ | 방향 불일치 ❌ | ❌ |
| FPI → 리뷰수 → 폐업 | p=0.413 ❌ | p=0.000 ✅ | ❌ |

별점 계수(coef=-0.300, OR=0.741)가 음수로 나타나 "별점↑ → 영업중확률↓"라는 직관과 반대 결과였다. 실제 데이터 확인 결과 폐업 브랜드 평균 별점(3.60)이 영업 중 브랜드(3.52)보다 오히려 높게 나타났다. is_open이 2017년 스냅샷이므로 현재 시점 별점이 폐업의 선행 지표로 작동하지 않는 횡단면 데이터의 구조적 한계다.

**결론**: 두 매개 경로 모두 통계적으로 성립하지 않음. 폐업의 핵심 변수는 리뷰수(OR=1.621, p<0.001).

⭐ **채용 관점**: 부정 결과의 원인을 데이터 구조(횡단면 한계)로 해석하고, "현재 별점이 폐업 선행 지표로 작동하지 않는다"는 데이터 특성을 직접 확인. 방법론적 한계를 명확히 진단한 분석 태도를 보여준다.

## 6. 프로젝트에서 직접 설계한 것

### 6-1. FPI (Franchise Pressure Index) — 핵심 차별화 요소 ⭐

기존 입지 분석은 주로 상권의 유동인구, 임대료, 경쟁 업체 수를 활용한다. 본 프로젝트는 이를 넘어 **경쟁 업체의 품질(별점)·인지도(리뷰수)·소비자 반응(감성점수)을 거리 감쇠로 가중합산**한 복합 지수를 설계했다.

- 거리 감쇠 함수 선택: 단순 개수 카운팅이 아닌 거리 역제곱 적용으로 공간적 영향력의 현실적 표현
- 민감도 분석으로 임계거리 데이터 기반 확정 (주관적 설정 배제)
- Shift 정규화 설계: 음수 제거 + 상대 간격 보존 동시 달성

### 6-2. TF-IDF 기반 도메인 특화 감성 사전

- 기성 VADER를 검증 도구로 활용하고, 직접 구축한 사전을 주분석에 사용
- 3점 리뷰 의도적 제외로 감성 사전 오염 방지 (데이터 설계적 결정)
- 교집합 제거로 순수 긍정/부정 사전 확보

### 6-3. 업종 범위 정의 기준

- Burgers + Sandwiches + Fast Food를 패스트푸드(QSR) 업종으로 묶은 근거 수립
- Pizza, Chicken 제외 근거: 소비 패턴(배달·가족 단위 수요)과 경쟁 구조의 차이

### 6-4. 생존/고전 브랜드 정의 기준

- HP 구간 내에서 별점 4.0 이상 + 리뷰 10개 이상 = 생존, 별점 3.0 이하 = 고전
- 리뷰수 최소 기준(10개)으로 통계적 신뢰도 확보
- 리뷰수 불균형 → 언더샘플링 검증 수행

### 6-5. 복수 분석 방법의 교차 검증 설계

단일 방법론에 의존하지 않고 TF-IDF / N-gram / VADER / LDA / 시계열 / 직접 언급 분석 6가지로 동일 가설을 독립 검증하는 프레임워크 설계.

## 9. 어려웠던 점

### 기술적 문제

**FPI 음수 문제**  
감성점수가 음수인 프랜차이즈로 인해 FPI가 음수가 되는 문제가 FPI 산출 단계에서 발견됐다. MinMax 정규화는 절대 크기를 잃고, 상수 추가는 임의적이라 Shift 정규화(최솟값 차감)를 선택했다. 이 선택이 "상대적 간격을 보존한다"는 이유를 명확히 설명할 수 있게 문서화해뒀다.

**하버사인 거리 행렬 연산**  
PART1에서 약 521만 쌍(4,818 × 1,081), PART2에서 약 62만 쌍(775 × 804)의 거리를 계산해야 했다. NumPy 벡터 연산으로 처리했다.

**매개효과 경로 불성립**  
PART2 로지스틱 회귀에서 별점 계수가 음수(OR=0.741)로 나타나 직관과 반대 결과가 도출됐다. 원인을 추적하니 폐업 브랜드 평균 별점(3.60)이 영업 중 브랜드(3.52)보다 높았다. is_open이 2017년 스냅샷이라는 횡단면 데이터의 구조적 한계임을 파악하고, 매개효과 경로가 성립하지 않는다는 결론을 내렸다.

### 데이터 문제

**브랜드명 표기 불일치**  
"McDonald's"와 "McDonalds"처럼 같은 브랜드가 다르게 표기되는 문제를 이름 정제(소문자화 + 특수문자 제거) 후 수동 매핑(13개 병합)으로 해결했다. 자동화가 어려운 작업이었고, FuzzyWuzzy 등 퍼지 매칭으로 개선할 수 있다는 점을 문서화했다.

## 11. 가장 의미 있었던 순간

### 1. FPI 음수 문제 발견과 해결

FPI를 처음 산출했을 때 음수가 나오는 경우를 발견했다. 단순히 이상값으로 제외하거나 절댓값을 취할 수도 있었지만, 원인을 추적해 "감성점수 음수"가 근본 원인임을 파악했다. Shift 정규화가 왜 최적 해결책인지 논리적으로 설명할 수 있게 됐다.

### 2. 고전 브랜드 시계열 분석에서 "한때는 좋았는데" 패턴 발견 ⭐

단순히 "고전 브랜드는 처음부터 나쁘다"고 예상했지만, 시계열 분석에서 2006~2008년에는 고전 브랜드가 생존 브랜드보다 높은 감성점수를 보인 구간이 있었다. 실제 리뷰로 검증하니 "this place has gone downhill from years ago. in the past it was really awesome... now it's cafeteria style"이 발견됐다. 처음부터 나쁜 게 아니라 **시간이 지나면서 품질 유지에 실패한 것**이라는 새로운 인사이트를 데이터로 직접 확인한 순간.

### 3. McDonald's가 최소 기준선이라는 발견

프랜차이즈 직접 언급 분석에서 McDonald's 언급 맥락을 실제 리뷰로 샘플링했을 때, 생존 브랜드와 고전 브랜드의 언급 방식이 정반대였다. "McDonald's보다 낫다"는 자랑과 "McDonald's가 나았을 것"이라는 실망. 소비자 인식의 기준선이 무엇인지를 질적으로 확인한 순간.

### 4. 반직관적 결과의 원인 탐색

FPI가 폐업에 직접 영향이 없고 별점 계수도 음수라는 반직관적 결과가 나왔을 때 "분석이 실패했다"고 해석하지 않고, 실제 데이터를 확인해 폐업 브랜드의 평균 별점(3.60)이 영업 중 브랜드(3.52)보다 높다는 사실을 발견했다. is_open이 스냅샷 데이터라는 구조적 한계를 직접 데이터로 확인하고, 폐업의 핵심 변수가 리뷰수라는 대안적 발견으로 연결된 과정이 분석의 완결성을 높였다.

## 12. 면접 활용 포인트

### STAR 사례 1 — 지표 직접 설계 ⭐

**Situation**: Yelp 데이터에는 프랜차이즈가 독립 레스토랑에 미치는 경쟁 압력을 나타내는 지표가 없었다.  
**Task**: 경쟁 압력을 정량화할 수 있는 지표를 팀이 직접 설계해야 했다.  
**Action**: 프랜차이즈의 별점·리뷰수·감성점수를 거리 감쇠로 가중합산하는 FPI 공식을 설계했다. 민감도 분석으로 임계거리(300m)를 데이터 기반으로 결정했고, 음수 문제 발생 시 Shift 정규화를 도입해 해결했다.  
**Result**: 두 분석(PART1/PART2)에서 FPI가 별점에 유의미한 부(-)의 영향을 미침을 통계적으로 검증했다. 업종을 좁히니 효과가 2.3배 강해져 지표의 타당성을 재확인했다.

---

### STAR 사례 2 — 분석 방향 전환 의사결정

**Situation**: PART1 텍스트 분석 도중 카지노 키워드가 결과를 오염시키고 업종 이질성이 FPI 구간 차이를 가리고 있음을 발견했다.  
**Task**: 방법론을 수정하거나 범위를 좁혀 더 정교한 분석을 수행할지 결정해야 했다.  
**Action**: 업종 이질성과 카지노 노이즈를 구체적 근거로 팀에 제시하고, 동일한 소비 구조를 공유하는 패스트푸드 3개 카테고리로 범위를 축소하는 PART2를 제안했다.  
**Result**: PART2에서 FPI 효과가 2.3배 강해지고, 텍스트 분석에서 "고전 브랜드의 도태 경로"와 같은 구체적 인사이트를 도출했다.

---

### STAR 사례 3 — 부정 결과와 데이터 구조 한계 발견 ⭐

**Situation**: FPI가 폐업에 직접 영향을 미치지 않는다는 결과(p=0.401)가 나왔다. 별점 계수도 음수(OR=0.741)로 나타나 "별점↑ → 폐업 확률↓"라는 직관과 반대 결과였다.  
**Task**: 이 반직관적 결과를 데이터 오류로 볼지, 구조적 한계로 볼지 판단해야 했다.  
**Action**: 실제 데이터를 확인하니 폐업 브랜드의 평균 별점(3.60)이 영업 중 브랜드(3.52)보다 오히려 높았다. is_open이 2017년 스냅샷이므로 현재 시점의 별점이 폐업의 선행 지표로 작동하지 않는 횡단면 데이터의 구조적 한계임을 파악했다.  
**Result**: 매개효과 두 경로 모두 성립하지 않는다는 결론. 폐업의 핵심 변수는 리뷰수(OR=1.621)임을 발견. 향후 패널 데이터 또는 생존 분석이 필요하다는 방법론적 한계를 명확히 제시했다.

---

### STAR 사례 4 — 복수 방법론 교차 검증

**Situation**: 텍스트 분석 결과만으로는 "생존 브랜드가 음식 품질이 높다"는 주장이 TF-IDF 방법론의 아티팩트일 수 있었다.  
**Task**: 결론의 신뢰도를 높이기 위한 추가 검증이 필요했다.  
**Action**: TF-IDF 차이, N-gram, VADER Intensity, LDA, 시계열, 프랜차이즈 직접 언급 6가지 독립 방법으로 동일 가설 검증.  
**Result**: 6가지 방법 모두 동일한 방향을 지시. "하나의 방법론으로만 도달한 결론이 아님"을 명확히 주장 가능.

## 14. 회고

### 잘한 점

1. **분석 근거의 명문화**: 모든 파라미터 결정(프랜차이즈 임계값, 임계거리, 통제변수 선택 등)에 근거를 기록했다. "왜 이렇게 했는가"에 항상 답할 수 있는 구조.
2. **부정 결과의 적극적 해석**: FPI → 폐업 직접 영향 없음 결과를 데이터 구조(횡단면 한계)로 해석하고, 폐업의 핵심 변수(리뷰수, OR=1.621)를 발견. 통계 결과를 기계적으로 읽지 않았다.
3. **복수 방법론 설계**: 단일 방법론의 한계를 인식하고 여러 방법으로 교차 검증. 분석 신뢰도에 대한 진지한 태도.
4. **PART2 독립 수행**: PART1의 구조적 한계를 발견하고 단독으로 PART2 전체를 설계·실행. 자기 주도적 분석 경험.
5. **실제 리뷰 샘플링**: 통계 결과를 정성적으로 검증. "McDonald's 기준선" 인사이트는 실제 리뷰를 읽지 않았으면 발견하지 못했을 것.

### 아쉬운 점

1. **R² 낮음 (0.08)**: 설명력이 낮다. 업종 클러스터링, 가격대, 좌석 수 등 추가 통제변수가 있었다면 더 강한 모델을 만들었을 것.
2. **시간 고정 효과 없음**: 리뷰는 2004~2022년에 걸쳐 있는데 연도별 고정 효과를 통제하지 않았다. 패널 데이터 구조로 접근했으면 역인과 가능성도 더 엄밀히 통제할 수 있었다.
3. **단일 도시 한계**: Las Vegas는 관광 도시라는 특수성이 강하다. Phoenix, Charlotte에도 동일 분석을 적용해 일반화 가능성을 검토하지 못했다.
4. **브랜드명 정제 자동화 부재**: FuzzyWuzzy 같은 퍼지 매칭을 적용했다면 수동 보정의 임의성을 줄일 수 있었다.
5. **횡단면 데이터 한계**: is_open이 2017년 스냅샷으로 시간적 선후관계를 검증할 수 없어 폐업 인과 경로 분석에 제약이 있었다. 패널 데이터나 생존 분석이 있었다면 더 엄밀한 검증이 가능했을 것이다.

### 다시 한다면 개선할 점

1. **ABSA (Aspect-Based Sentiment Analysis)**: 음식/서비스/가격/분위기를 차원별로 분리 분석해 "무엇이 생존을 결정하는가"를 더 세밀하게 보고 싶다.
2. **BERTopic / word2vec**: 임베딩 기반 토픽 모델로 LDA보다 의미론적으로 풍부한 클러스터링.
3. **공간 자기상관 보정**: Moran's I 검정으로 FPI 공간적 자기상관을 통제한 회귀분석.
4. **시계열 패널 데이터 구조**: 연도별 FPI 변화와 별점 변화를 연계해 인과관계를 더 엄밀히 검증.
5. **Plotly Dash 통합 대시보드**: 현재 HTML 지도 9개가 개별 파일로 분리되어 있어 통합 대시보드로 묶었으면 실무 활용성이 높아졌을 것.

### 3문장 요약

Yelp 공개 데이터(업체 17만 개, 리뷰 526만 개)를 활용해 Las Vegas 독립 레스토랑이 주변 프랜차이즈로부터 받는 경쟁 압력을 정량화하는 FPI(Franchise Pressure Index) 지수를 팀이 직접 설계했다. FPI가 별점에 유의미한 부(-)의 영향을 미치며(β=-0.027), 폐업에 대한 직접 효과는 없음(p=0.401)을 검증했고, 매개효과 분석은 횡단면 데이터의 구조적 한계로 성립하지 않았으며 폐업의 핵심 변수는 리뷰수(OR=1.621)임을 확인했다. 6가지 텍스트 분석 방법을 통해 HP 구간 생존 브랜드는 시그니처 메뉴와 수제·오너십 강조로 차별화하는 반면, 고전 브랜드는 초기 차별화 이후 품질 유지 실패와 저가화로 도태되는 경로를 데이터로 확인했다.

### 1분 자기소개 버전

"저는 Yelp 데이터를 활용한 프랜차이즈 경쟁압력 분석 프로젝트를 수행했습니다. 이 프로젝트의 핵심 과제는 '독립 레스토랑이 주변 프랜차이즈로부터 받는 압력을 어떻게 숫자로 표현할 것인가'였습니다. 기존에 이를 측정하는 지표가 없었기 때문에, 저희 팀은 FPI라는 지수를 직접 설계했습니다. 프랜차이즈의 별점·리뷰수·감성점수를 거리 감쇠로 가중합산하는 방식이었고, 임계거리는 민감도 분석으로 데이터 기반으로 결정했습니다. 분석 과정에서 FPI가 폐업에 직접 영향이 없다는 결과가 나왔는데, 별점 계수가 음수로 나타나는 반직관적 결과의 원인을 추적하니 is_open이 2017년 스냅샷이라는 횡단면 데이터 구조적 한계임을 확인했습니다. 폐업의 실제 핵심 변수는 리뷰수(OR=1.621)였습니다. 또한 텍스트 분석을 6가지 방법으로 교차 검증해 '고전 브랜드는 처음부터 나쁜 게 아니라 시간이 지나면서 품질 유지에 실패했다'는 인사이트를 데이터로 확인했습니다. 이 프로젝트를 통해 기존에 없는 지표를 설계하는 것, 부정 결과를 데이터 구조로 해석하는 것, 단일 방법론에 의존하지 않는 것이 데이터 분석에서 중요하다는 것을 배웠습니다."
